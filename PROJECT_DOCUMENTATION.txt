SECTION 3: DATABASE MODELS & SCHEMA
================================================================================

1. USER MODEL (models/User.js)
   Fields:
   - name (String, required, trimmed) - User's display name
   - email (String, required, unique, lowercase) - User's email address
   - password (String, required) - Hashed password
   - role (String, enum: ['visitor', 'creator', 'admin'], default: 'visitor')
   - bio (String, default: '') - User biography
   - profilePicLink (String, default: '') - Profile picture URL
   - isBanned (Boolean, default: false) - Account suspension status
   - timestamps: true - Auto createdAt, updatedAt fields
   
   Methods:
   - matchPassword(enteredPassword) - Compare password for login
   - Hooks: pre-save middleware for password hashing

2. EXHIBIT MODEL (models/Exhibit.js)
   Fields:
   - title (String, required, trimmed) - Artwork title
   - description (String, required) - Artwork description
   - category (String, enum: ['Art', 'Photography'], required)
   - mediaLink (String, required) - URL to image/video file
   - mediaType (String, enum: ['image', 'video'], default: 'image')
   - creator (ObjectId, ref: 'User', required) - Creator's user ID
   - status (String, enum: ['pending', 'approved', 'rejected'], default: 'pending')
   - likes (Array of ObjectId, ref: 'User') - Array of user IDs who liked
   - shareCount (Number, default: 0) - Number of times shared
   - tags (Array of String) - Search and categorization tags
   - timestamps: true - Auto createdAt, updatedAt

3. COMMENT MODEL (models/Comment.js)
   Fields:
   - exhibit (ObjectId, ref: 'Exhibit', required) - Parent exhibit ID
   - author (ObjectId, ref: 'User', required) - Comment author's user ID
   - text (String, required, trimmed) - Comment content
   - timestamps: true - Auto createdAt, updatedAt

SECTION 4: API ROUTES & ENDPOINTS
================================================================================

AUTH ROUTES (routes/auth.js):
1. POST /api/auth/register
   - Body: { name, email, password, role }
   - Response: { message, token, user }
   - Use: Register new user account

2. POST /api/auth/login
   - Body: { email, password }
   - Response: { message, token, user }
   - Use: Authenticate user and get JWT token

EXHIBITS ROUTES (routes/exhibits.js):
1. GET /api/exhibits
   - Query: { category, search, page, limit }
   - Response: { exhibits, total, page, pages }
   - Use: Browse all approved exhibits with filtering and pagination
   - Filters: Status='approved' only, search by title/description/tags, category filter

2. GET /api/exhibits/:id
   - Response: Exhibit object with populated creator info
   - Use: Get single exhibit detail page

3. POST /api/exhibits (Protected, Creator+)
   - Body: { title, description, category, mediaLink, mediaType, tags }
   - Response: { message, exhibit }
   - Use: Create new exhibit submission
   - Status: Auto-set to 'pending'

4. PUT /api/exhibits/:id (Protected, Creator+)
   - Body: { title, description, category, mediaLink, mediaType, tags }
   - Use: Edit own exhibit (resets status to pending for re-approval)
   - Auth: Creator or owner verification

5. DELETE /api/exhibits/:id (Protected)
   - Use: Delete exhibit (creator or admin only)
   - Cascades: Also deletes all comments on exhibit

6. POST /api/exhibits/:id/like (Protected)
   - Use: Toggle like on exhibit
   - Response: { likes (count), liked (boolean) }

7. POST /api/exhibits/:id/comment (Protected)
   - Body: { text }
   - Response: Comment object with populated author info
   - Use: Add comment to exhibit

USERS ROUTES (routes/users.js):
1. GET /api/users/profile (Protected)
   - Response: { user, exhibits (creator's exhibits) }
   - Use: Get current user's profile and their exhibits

2. PUT /api/users/profile (Protected)
   - Body: { name, bio, profilePicLink }
   - Response: { message, user }
   - Use: Update own profile

3. GET /api/users/:id (Public)
   - Response: { user (public info only), exhibits (approved only) }
   - Use: View public profile of another user

ADMIN ROUTES (routes/admin.js):
1. GET /api/admin/stats (Protected, Admin only)
   - Response: { totalUsers, totalExhibits, pendingExhibits, approvedExhibits, rejectedExhibits, totalComments }
   - Use: Dashboard statistics

2. GET /api/admin/users (Protected, Admin only)
   - Response: Array of all users (password excluded)
   - Use: User management page

3. PUT /api/admin/users/:id (Protected, Admin only)
   - Body: { role, isBanned }
   - Response: { message, user }
   - Use: Change user role or ban status

4. DELETE /api/admin/users/:id (Protected, Admin only)
   - Use: Delete user and cascade delete their exhibits
   - Prevents: Self-deletion

5. GET /api/admin/exhibits (Protected, Admin only)
   - Query: { status } (optional)
   - Response: Array of all exhibits with specified status
   - Use: Browse all exhibits by status

6. PUT /api/admin/exhibits/:id/approve (Protected, Admin only)
   - Use: Approve pending exhibit
   - Sets: status = 'approved'

7. PUT /api/admin/exhibits/:id/reject (Protected, Admin only)
   - Use: Reject pending exhibit
   - Sets: status = 'rejected'

8. DELETE /api/admin/exhibits/:id (Protected, Admin only)
   - Use: Delete exhibit and cascade delete comments

SECTION 5: MIDDLEWARE
================================================================================

FILE: middleware/auth.js

1. protect(req, res, next)
   - PURPOSE: Verify JWT token and authenticate user
   - CHECKS: Authorization header with Bearer token
   - SETS: req.user object with decoded token data { id, role, name }
   - RETURNS: 401 if no token or invalid token

2. creatorOnly(req, res, next)
   - PURPOSE: Restrict route to creators and admins only
   - CHECKS: req.user.role must be 'creator' or 'admin'
   - RETURNS: 403 if unauthorized

3. adminOnly(req, res, next)
   - PURPOSE: Restrict route to admins only
   - CHECKS: req.user.role must be 'admin'
   - RETURNS: 403 if unauthorized
